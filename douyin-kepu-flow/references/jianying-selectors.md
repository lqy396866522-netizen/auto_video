# 剪映 UI 自动化选择器

> 剪映 11.x 桌面版 · Windows · **纯坐标点击**（QML 渲染，UIA 树为空，名称匹配无效）

## 关键结论（已验证）

- 剪映 11.x 窗口 UIA `descendants()` 返回 **0 个元素** → 无法用名称/控件定位，全部走 `pyautogui` 坐标
- 运行前必须 `bring_to_front(win)` 置顶剪映（`SetForegroundWindow` + AttachThreadInput），否则微信等窗口会遮挡导入区
- 多屏：窗口可能在副屏（origin x>0），坐标用 `win.rectangle()` 的 left/top + 相对比例换算
- 截图验证用 `PIL.ImageGrab.grab(bbox=..., all_screens=True)`，否则副屏返回全黑

## 文件对话框（标准 Windows #32770）

- 标题「请选择媒体资源」，用 `win32` 后端
- Alt+D 跳转到 VideoDir → 文件名(N) 粘贴带引号文件名 → 打开(O)

## 前置条件

- 剪映在**首页**空闲（无未保存编辑弹窗）
- 视频目录内已有 **10 个 mp4**，文件名含 **01–10** 序号
- 运行脚本时勿操作鼠标键盘

## 流程步骤

| 步 | 动作 | 定位方式 |
|----|------|----------|
| 1 | 启动/附着剪映主窗口 | `title_re=.*剪映.*` |
| 2 | 点击 **开始创作** | `name=开始创作` Button |
| 3 | 点击 **+ 导入** | `name=导入` 或含「导入」Button |
| 4 | 文件对话框选 10 个 mp4 | 标题 **「请选择媒体资源」**；**先 Alt+D 跳转到 `-VideoDir` 指定目录**（对话框默认在「下载」无影响），再选 `seg-01..10` → **打开(O)** |
| 5 | 循环 10 次点素材 **⊕** | ListItem 内 `name=+` 或坐标 fallback |
| 6 | 顶栏 **字幕** | `name=字幕` |
| 7 | **新建字幕** | `name=新建字幕` |
| 8 | **手动写字幕** | `name=手动写字幕` |
| 9 | 粘贴 `narration_script` | 剪贴板 Ctrl+V |
| 10 | **配音选择** | `name=配音选择` |
| 11 | 侧栏 **收藏** → **活力科普** | Tab + ListItem |
| 12 | **添加到轨道** | `name=添加到轨道` |
| 13 | 等待 **自动拆句中…** 消失 | 轮询弹窗文案，超时 300s |

## 固定参数

| 项 | 值 |
|----|-----|
| 配音音色 | 活力科普 |
| 配音 Tab | 收藏 |
| 智能分割字幕 | 保持勾选 |

## 视频目录 CLI

```powershell
-VideoDir "D:\my-clips\buffet"           # 绝对路径
-VideoDir "buffet-profit-secret"        # → D:\douyin-videos\{name}
```

## 故障排查

- **找不到按钮**：剪映更新后改文案，更新本文件并重跑
- **导入失败**：对话框标题为「请选择媒体资源」，用 `win32` 后端 + 文件名(N) 输入框粘贴带引号的完整路径；目录内有多余 mp4 时用 `-ImportOneByOne`
- **+ 号点错序**：确保导入时按 01–10 顺序多选；或改用 `--import-one-by-one`
